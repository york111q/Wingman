Welcome to Wingman project


How it works:

  1. Match results along with statistics can be found on the official Steam website, but they are only available after logging in, which makes it impossible to scrape data directly from there.
  2. After a game session, I save the web page as an .html file and upload it to the form available at "New matches".
  3. The script, using BeautifulSoup, scrapes data on new matches and saves it in a temporary .txt file based on which a new form is created where I can select matches that were played outside of pools and should not be included in the database.
  4. After verification the matches are added either to the list of matches outside of pools or to the database.
  5. When entering the summary or summary 20 page, another script creates a summary using data aggregation.
  6. The list of matches and the ranking of pools is displayed using ListView.
  7. The project is hosted on Heroku and therefore the database used is PostgreSQL.

Rules of play:

  1. Out of all the maps available in the Wingman CS:GO queue, we play 7 different ones. We call such a set of 7 matches a 'pool'.
  2. After each pool, maps with the worst round ratio are chosen to be skipped in the next pool so that each pool has 7 maps. The only exception is when we lose a match 0:9, in which case this map will be skipped in the next pool regardless of the round ratio.
  3. You cannot skip maps that were skipped in the previous pool.
  4. Two maps, currently: Calavera, Pitstop, were recently added to the game and we decided that in each pool we will play one of them, skip the other and complete the pool with 6 maps from the others. The matchmaking for CS:GO randomly decides which of these two maps will appear in the pool.
